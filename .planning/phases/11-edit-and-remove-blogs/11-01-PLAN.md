<task type="auto">
  <name>Task 2: Create HTMX edit and display handlers</name>
  <files>internal/server/handlers.go, internal/server/routes.go</files>
  <action>
In handlers.go, add three handlers after handleAddBlog (around line 500):

1. **handleGetBlog(w http.ResponseWriter, r *http.Request)**
   - Extract blog ID from URL path: `/{id}`
   - Parse ID using strconv.ParseInt
   - Call db.GetBlogByID(id)
   - Handle errors: 404 if blog not found, 500 for database errors
   - Query article count for this blog: `SELECT COUNT(*) FROM articles WHERE blog_id = ?`
   - Execute template "blog-display-row.gohtml" with blog data and article count
   - Template data: struct with Blog field and ArticleCount field
   - Returns HTML display row partial (HTMX swap target for cancel button)

2. **handleEditBlog(w http.ResponseWriter, r *http.Request)**
   - Extract blog ID from URL path: `/{id}/edit`
   - Parse ID using strconv.ParseInt
   - Call db.GetBlogByID(id)
   - Handle errors: 404 if blog not found, 500 for database errors
   - Execute template "blog-edit-form.gohtml" with blog data
   - Template data: struct with Blog field
   - Returns HTML form partial (HTMX swap target)

3. **handleUpdateBlogName(w http.ResponseWriter, r *http.Request)**
   - Extract blog ID from URL path
   - Parse form: r.ParseForm()
   - Get new name: r.FormValue("name")
   - Validate: name must be 1-100 chars (return 400 if invalid)
   - Call db.UpdateBlogName(id, name)
   - On success: query updated blog with GetBlogByID
   - Query article count for this blog
   - Execute template "blog-display-row.gohtml" with updated blog data and article count
   - Add response header: `HX-Trigger: blogListUpdated` (triggers sidebar refresh)
   - Returns HTML blog card partial (replaces edit form)

In routes.go, add three routes in registerRoutes after POST /blogs/add:
- `GET /blogs/{id}` -> handleGetBlog (for cancel button - returns display row)
- `GET /blogs/{id}/edit` -> handleEditBlog (for edit button - returns edit form)
- `PUT /blogs/{id}` -> handleUpdateBlogName (for save button - returns display row)

Use existing pattern from settings route (line 23) for path parameters.
