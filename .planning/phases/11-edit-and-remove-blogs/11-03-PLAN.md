Add two deletion methods to Database struct (after UpdateBlog method, around line 494):

1. **DeleteBlogOnly(id int64) error**
   - Execute in transaction: BEGIN
   - Update articles: `UPDATE articles SET blog_id = NULL WHERE blog_id = ?` (orphan articles)
   - Delete blog: `DELETE FROM blogs WHERE id = ?`
   - COMMIT transaction
   - Return error if any step fails (transaction rolls back)

2. **DeleteBlogWithArticles(id int64) error**
   - Execute in transaction: BEGIN
   - Delete articles first: `DELETE FROM articles WHERE blog_id = ?` (cascade delete)
   - Delete blog: `DELETE FROM blogs WHERE id = ?`
   - COMMIT transaction
   - Return error if any step fails (transaction rolls back)
   - Note: FTS5 DELETE trigger handles articles_fts cleanup automatically

Both methods use transactions to ensure atomicity. If blog deletion fails, article operations roll back.
