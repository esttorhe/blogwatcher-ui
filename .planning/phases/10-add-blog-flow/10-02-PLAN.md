  <what-built>
Complete add blog flow: form submission with CLI integration, error handling, auto-sync, and FAB quick access
  </what-built>
  <how-to-verify>
**Prerequisites:**
- Ensure `blogwatcher` CLI is installed and in PATH: `which blogwatcher`
- Start server: `go run ./cmd/server` or `./blogwatcher-ui`
- Navigate to http://localhost:8080

**Test 1: Add Blog via Settings Page**
1. Click gear icon in sidebar to navigate to settings
2. Verify "Add New Blog" form appears at top of settings page
3. Enter test blog:
   - Name: "Test Blog"
   - URL: "https://blog.golang.org"
4. Click "Add Blog" button
5. Verify loading indicator appears ("Adding...")
6. Wait for response (may take 5-30 seconds for feed discovery)
7. Verify success message: "✓ Successfully added 'Test Blog'. Fetching articles..."
8. Click "Back to Settings" button
9. Verify "Test Blog" appears in "Tracked Blogs" list with article count
10. Navigate back to article list (click "Inbox" in sidebar)
11. Wait 10-15 seconds for auto-sync to complete
12. Verify articles from "Test Blog" appear in article list

**Test 2: FAB Quick Access**
1. From article list page, locate FAB (floating + button) in bottom-right corner
2. Verify FAB is visible and styled correctly (circular, accent color, drop shadow)
3. Click FAB button
4. Verify settings page loads via HTMX (no full page reload)
5. Verify "Add New Blog" form is focused (cursor in name field)

**Test 3: Error Handling - Duplicate Blog**
1. Navigate to settings page
2. Try to add the same blog again:
   - Name: "Test Blog"
   - URL: "https://blog.golang.org"
3. Click "Add Blog"
4. Verify error message appears: "Blog with name 'Test Blog' already exists" (or similar)
5. Verify form fields are pre-populated with values entered (no re-typing needed)

**Test 4: Error Handling - Invalid URL**
1. Clear form fields
2. Enter:
   - Name: "Invalid Blog"
   - URL: "not-a-valid-url"
3. Click "Add Blog"
4. Verify error message appears about invalid URL or feed not found
5. Verify form fields are pre-populated

**Test 5: Error Handling - No Feed Found**
1. Clear form fields
2. Enter:
   - Name: "No Feed Blog"
   - URL: "https://google.com"
3. Click "Add Blog"
4. Verify error message appears about no RSS/Atom feed found
5. Verify form fields are pre-populated

**Test 6: Mobile Responsiveness**
1. Resize browser to mobile width (<768px) or use mobile device
2. Verify FAB is smaller (48px instead of 56px)
3. Verify FAB position adjusted (1rem from edges instead of 2rem)
4. Verify form inputs are full width and readable
5. Click FAB from mobile view
6. Verify navigation works and form is usable

**Test 7: Loading States**
1. Add a blog with slow feed discovery (try a blog with slow server)
2. Verify loading indicator ("Adding...") appears immediately
3. Verify button is disabled during submission (prevent double-submit)
4. Verify timeout after 30 seconds if feed discovery hangs

**Test 8: Auto-Sync Verification**
1. Check server logs after successful blog add
2. Verify log message: "Added blog 'X' at URL"
3. Verify log message: "Auto-synced X: N new articles"
4. Verify articles appear in article list within 15-20 seconds

**Test 9: Browser Back Button**
1. Add blog successfully
2. Click browser back button
3. Verify navigation works correctly (returns to previous view)
4. Click forward button
5. Verify settings page loads with updated blog list

**Test 10: Direct URL Access**
1. Navigate directly to http://localhost:8080/settings
2. Verify full page loads with sidebar and add blog form
3. Test form submission from direct access
4. Verify HTMX updates work correctly

**Cleanup:**
After testing, remove test blog using CLI:
```
blogwatcher remove "Test Blog"
```
